02 LLM-Router-Proxy v3.3 LLM Router Proxy v3.3

FastAPI-Middleware auf Port 11435 (~/llm_router/llm_proxy.py). Jede Anfrage von Open WebUI und allen anderen Clients läuft ausschließlich über diesen Proxy.

Zweistufiges Routing:

  • Stufe 1 — Regelbasierter Dispatcher: Domain-Keywords, Token-Länge, Themenklassifikation. Kein LLM-Call, Entscheidung in <1 ms.
  • Stufe 2 (ambig) — L1-Reformulator: Ein kleineres LLM klassifiziert die Absicht, empfiehlt das Modell, extrahiert Such-Queries.

Anti-Halluzinations-Protokoll: P4-Struktur (Prior Identification → Causal Mechanisms → Counter-evidence → Judgement) in allen epistemischen Prompts. Synthegy-Evaluator: Jede Antwort wird durch ein separates Evaluator-LLM bewertet (Genauigkeit 1–5, Fundierung 1–5) bevor Auslieferung.

Spezialbefehle: /ensemble, /kreativ, /navigator, /destilliere, /speichere, /journal — epistemische Orchestrierungs-Kommandos.

FastAPI middleware on port 11435 (~/llm_router/llm_proxy.py). Every request from Open WebUI and all other clients passes exclusively through this proxy.

Two-stage routing:

  • Stage 1 — Rule-based dispatcher: domain keywords, token length, topic classification. No LLM call, decision in <1 ms.
  • Stage 2 (ambiguous) — L1 reformulator: A smaller LLM classifies intent, recommends model, extracts search queries.

Anti-hallucination protocol: P4 structure (Prior Identification → Causal Mechanisms → Counter-evidence → Judgement) in all epistemic prompts. Synthegy evaluator: Every response scored by a separate evaluator LLM (accuracy 1–5, groundedness 1–5) before delivery.

Special commands: /ensemble, /kreativ, /navigator, /destilliere, /speichere, /journal — epistemic orchestration commands.
