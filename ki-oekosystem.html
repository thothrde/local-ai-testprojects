lean_iterativ v4 — Iterativer Proof-Refinement-Loop

Open-WebUI-Tool für iterative Lean 4-Beweisverifikation auf Basis des Ralph Loop (AlphaProof Nexus, arXiv:2605.22763). Das Modell deepseek-r1:70b sieht Compiler-Feedback und korrigiert gezielt, bis alle sorry-Platzhalter durch verifizierte Beweise ersetzt sind.

Implementierungsgrundlage: Li et al. „Compile to Compress: Boosting Formal Theorem Provers by Compiler Outputs" (ICML 2026). Vier Verbesserungen gegenüber dem ursprünglichen Ralph Loop:

  • A — Inline-Fehlerinjektion: Compiler-Fehler werden direkt an der betreffenden Codezeile als -- ERROR:-Kommentar eingefügt, statt am Ende angehängt. Li et al. Table 10: +3 % auf MiniF2F gegenüber Append-Strategie.
  • B — Markovianische Disziplin: War bereits korrekt im Original. Jede Runde erhält ausschließlich (aktueller Code + Compiler-Fehler) — keine akkumulierte Gesprächshistorie.
  • C — Fehlertyp-Routing: Klassifikation nach Li et al. Table 7: unfixierbar (linarith failed, omega, nested error, max recursion depth — Frühabbruch ab Runde 2), schwer (unsolved goals, type mismatch — geringe, aber >0 % Korrektierbarkeit), reparierbar (failed_to_synthesize ~4 %, unknown identifier ~1,4 %).
  • D — BFS/DFS-Balance + EVOLVE-BLOCK: Parameter frische_versuche=N streut alle N Runden einen unabhängigen Direktversuch ein (Random Tree Search nach Li et al.). locked_regions=True schützt die Theorem-Signatur vor LLM-Modifikation (EVOLVE-BLOCK, AlphaProof Nexus).
  • v4 — Verifikations-Bugfix: Lake ignoriert neu erstellte .lean-Dateien (statische Modulliste nach erstem Build). Fix: _lean_kompiliere() überschreibt Test.lean temporär und stellt sie nach jedem Build exakt wieder her. JSON-Fehlerformat bei capture_output=True (--json-Flag) wird korrekt geparst.

Testergebnisse (5/5): Korrekter Beweis direkt erkannt (10 s) · sorry → LLM-Fix → Lean-Verifikation (93 s) · Falscher Beweis ohne sorry korrekt abgelehnt (77 s) · Ringidentität (a+b)(a−b) = a²−b² via ring-Taktik (212 s) · Log ohne Trace-Kontamination (64 s).

lean_iterativ v4 — Iterative Proof Refinement Loop

Open WebUI tool for iterative Lean 4 proof verification based on the Ralph Loop (AlphaProof Nexus, arXiv:2605.22763). The model deepseek-r1:70b sees compiler feedback and corrects iteratively until all sorry placeholders are replaced by verified proofs.

Implementation basis: Li et al. "Compile to Compress: Boosting Formal Theorem Provers by Compiler Outputs" (ICML 2026). Four improvements over the original Ralph Loop:

  • A — Inline error injection: Compiler errors inserted directly at the relevant code line as -- ERROR: comments, rather than appended at the end. Li et al. Table 10: +3% on MiniF2F versus append strategy.
  • B — Markovian discipline: Already correct in the original. Each round receives only (current code + compiler errors) — no accumulated conversation history.
  • C — Error-type routing: Classification per Li et al. Table 7: unfixable (linarith failed, omega, nested error, max recursion depth — early abort from round 2 onward), hard (unsolved goals, type mismatch — low but >0% repairability), repairable (failed_to_synthesize ~4%, unknown identifier ~1.4%).
  • D — BFS/DFS balance + EVOLVE-BLOCK: Parameter frische_versuche=N interleaves an independent fresh attempt every N rounds (Random Tree Search per Li et al.). locked_regions=True protects the theorem signature from LLM modification (EVOLVE-BLOCK, AlphaProof Nexus).
  • v4 — Verification bugfix: Lake ignores newly created .lean files (static module list after first build). Fix: _lean_kompiliere() temporarily overwrites Test.lean and restores it exactly after every build. JSON error format under capture_output=True (--json flag) correctly parsed.

Test results (5/5): Correct proof recognized directly (10s) · sorry → LLM fix → Lean verification (93s) · Wrong proof without sorry correctly rejected (77s) · Ring identity (a+b)(a−b) = a²−b² via ring tactic (212s) · Log trace-free (64s).

Signature

lean_iterativ(aufgabe_oder_sketch, modell='deepseek-r1:70b', max_runden=8, frische_versuche=0, locked_regions=True)
